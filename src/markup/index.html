Layzr.js

Download

A small, fast, modern, and dependency-free library for lazy loading images.

Small

Fast

Dep Free

Lazy loading boosts page speed by deferring the loading of images until they're in (or near) the viewport. This library makes it completely painless - maximizing speed by keeping options to a minimum.

The scroll event has been debounced to minimize the weight on the browser, and callbacks are executed within rAF to ensure optimal revealing.

Usage

Load the script:

<script src="layzr.min.js"></script>

For each image, put a placeholder in the src attribute, put the real image in the data-layzr attribute, and the retina image in the data-layzr-retina attribute.

<img src="optional/placeholder" data-layzr="normal/image" data-layzr-retina="optional/retina/image">

Create a new instance with your desired options. Defaults shown below:

var layzr = new Layzr({ attr: 'data-layzr', retinaAttr: 'data-layzr-retina', callback: null });

Options

attr / retinaAttr

Customize the data attributes Layzr gets image sources from when creating your instance of Layzr (omit the "data-"):

var layzr = new Layzr({   attr: 'custom-image-attribute',   retinaAttr: 'custom-retina-attribute' });

callback

Layzr can optionally fire a callback each time an image is loaded. This makes it easy to fire CSS3 or JS based animations once an image has loaded.

var layzr = new Layzr({   callback: function() {     // "this" refers to the image node     this.classList.add('animate');   } });

Browsers

Layzr natively supports IE10+.

It can easily be extended to support IE9 by providing a shim for requestAnimationFrame, as shown below:

// credit where it’s due: http://www.paulirish.com/2011/requestanimationframe-for-smart-animating/ window.requestAnimFrame = (function() {   return window.requestAnimationFrame ||          window.webkitRequestAnimationFrame ||          window.mozRequestAnimationFrame ||          function(callback) {            window.setTimeout(callback, 1000 / 60);          }; })();

There are currently no plans to include a shim/polyfill in the library, in the interest of file size.
