Layzr.js

A small, fast, modern, and dependency-free library for lazy loading.

Small

Fast

Lazy loading boosts page speed by deferring the loading of images until they're in (or near) the viewport. This library makes it completely painless - maximizing speed by keeping options to a minimum.

The scroll event has been debounced to minimize the weight on the browser, and images are revealed within rAF to ensure optimal rendering.

Usage

Load the script:

<script src="layzr.js"></script>

For each image, put a placeholder in the src attribute, the regular image in the data-layzr attribute, and the retina image in the data-layzr-retina attribute.

Note that images without a placeholder may have no height, or appear broken, before they're loaded.

<img src="optional/placeholder" data-layzr="normal/image" data-layzr-retina="optional/retina/image">

Create a new instance with your desired options. Defaults shown below:

var layzr = new Layzr({   selector: '[data-layzr]',   attr: 'data-layzr',   retinaAttr: 'data-layzr-retina',   bgAttr: 'data-layzr-bg',   threshold: 0,   callback: null });

Options

selector

Customize the selector Layzr uses to find elements to lazy load.

var layzr = new Layzr({   selector: '.custom-selector' });

attr / retinaAttr

Customize the data attributes Layzr gets image sources from when creating your instance of Layzr. Omit the “data-” part.

Images without either attribute will be ignored by Layzr.

var layzr = new Layzr({   attr: 'custom-image-attribute',   retinaAttr: 'custom-retina-attribute' });

bgAttr

Elements that match the selector and have this attribute will load their image as a background image.

var layzr = new Layzr({   bgAttr: 'custom-background-attribute' });

threshold

Adjust when images load using the threshold option. Positive values make the images load sooner.

Note that the threshold is a percentage of the viewport height - think of it as similar to the CSS vh unit.

// load images when they're half the viewport height away from the bottom of the viewport var layzr = new Layzr({   threshold: 50 });

callback

Layzr can invoke a callback function each time an image is loaded.

Note that Layzr does not check if the image is fully loaded before calling the function. Detecting image load is inconsistent at best in modern browsers.

// in the callback function, "this" refers to the image node var layzr = new Layzr({   callback: function() {     this.classList.add('class-name');   } });

Browsers

Layzr natively supports IE10+.

It can easily be extended to support IE9 by including a shim for requestAnimationFrame, as shown below:

// credit where it’s due: http://www.paulirish.com/2011/requestanimationframe-for-smart-animating/ window.requestAnimFrame = (function() {   return window.requestAnimationFrame ||          window.webkitRequestAnimationFrame ||          window.mozRequestAnimationFrame ||          function(callback) {            window.setTimeout(callback, 1000 / 60);          }; })();

There are currently no plans to include a shim/polyfill in the library, in the interest of file size.
